### 🎯 第四阶段：核心业务流转与“模拟”逻辑 开发计划

#### 第一步：后端基础盘面加固（前置校验与实体扩展）

- **目标**：确保数据流转前不发生冲突，满足需求书的限制条件。
- **核心任务**：
  1. **新增任务拦截**：在 `SysUavTaskService` 的新增方法中，校验绑定的设备是否处于“正常”状态，航线是否被其他未完成的任务占用，以满足系统互斥性要求。
  2. **时间校验**：前端拦截配合后端拦截，确保任务开始时间不可选择过去时间。
  3. **数据快照准备**：确认 `sys_uav_result` 表已准备好冗余字段（如任务名、设备名、执行人等），防止日后基础数据删除导致结果页“开天窗”。

#### 第二步：后端核心状态机与事务流转（业务大脑）

- **目标**：实现任务的“开始”、“完成”、“取消”逻辑，并与设备状态和结果表强联动。
- **核心任务**：
  1. **开始任务 (`startTask`)**：通过 `@Transactional` 事务，将任务状态改为“执行中”，并自动将关联设备状态切换为“任务中”。同时记录实际开始时间（用于后续算力计算）。
  2. **取消任务 (`cancelTask`)**：将任务状态改为“已取消”，设备状态自动恢复为“正常”，并且**不生成**巡防结果。
  3. **完成任务 (`completeTask`)**：
     - 任务状态改为“已完成”，进度设为100%。
     - 设备状态恢复为“正常”。
     - **结果自动生成闭环**：提取任务、设备、航线信息的快照，向 `sys_uav_result` 表插入一条记录，并附带默认的“巡防概述”文本。

#### 第三步：前端页面状态渲染与权限控制（用户交互）

- **目标**：前端按钮和显示根据后端数据状态进行动态变换，防止用户误操作。
- **核心任务**：
  1. **操作按钮动态显隐**：
     - “待执行”显示：开始任务、取消任务。
     - “执行中”显示：模拟监控、完成任务、取消任务。
     - “已完成/已取消”显示：删除。
  2. **列表进度条**：在任务列表页增加基于 Element Plus 的 `<el-progress>` 组件，直观展示每个任务的当前进度。

#### 第四步：前端高德地图“全真模拟”看板（答辩高光点）

- **目标**：利用高德地图 API 和前端定时器，完成需求书中最具视觉冲击力的功能。
- **核心任务**：
  1. **进度算力引擎**：编写 JS 逻辑，根据任务的 `实际开始时间` 和航线的 `预计巡防时长`，实时计算出“已完成百分比”和“预计剩余时间”。
  2. **轨迹回放动画**：在“模拟监控”弹窗中加载高德地图，解析航线的 JSON 点位数据。利用高德地图的 `Marker.moveAlong` 功能或定时刷新坐标，让无人机图标在地图上沿着既定路线平滑移动。
