## 三、已发现的问题/风险清单

| 编号 | 问题 | 严重程度 | 影响范围 | 建议 |
|------|------|---------|---------|------|
| P1 | 设备状态枚举不一致：需求书含"离线"，数据库无 | 低 | 答辩说明 | 统一口径：定义 4 种状态(正常/维修中/已报废/任务中)，解释离线为业务概念 |
| P2 | AI 识别仅为图片 URL 展示，非真正 AI 框取 | 中 | 3.5.3 | 如需真正 AI，需接入识别服务；当前满足演示基本要求 |
| P3 | `pauseTask` 接口缺少 `@PreAuthorize` 权限注解 | 低 | 安全 | 补充 `@PreAuthorize("@ss.hasPermi('uav:task:edit')")` |
| P4 | 任务完成自动归档时未填充 `findings`/`handlingInfo`/`aiImageUrl` | 低 | 数据完整性 | 当前设为 null 符合"默认空"逻辑，答辩时可说明后续编辑补充 |
| P5 | 删除任务时不会级联清理关联结果记录 | 低 | 数据一致性 | 产生孤儿数据，建议在 `deleteSysUavTaskByTaskIds` 中增加关联结果清理 |
| P6 | 设备/航线名称冗余字段(`equipmentName`/`routeName`)新建时未回填 | 低 | 数据一致性 | 任务表中 `equipmentName`/`routeName` 冗余字段存为空，实际展示靠前端关联查询 |
| P7 | 航线名称无唯一性校验 | 低 | 用户体验 | 建议增加航线名称去重校验（类似设备编号唯一校验） |
| P8 | 无任务进度自动递增机制 | 低 | 模拟真实性 | 进度全靠前端 `updateProgress` 手动同步，无后端定时递增；答辩时前端动画已覆盖此需求 |

## 七、总结

### 已完成的核心能力

1. **完整的业务闭环**：设备 → 航线 → 任务 → 结果，增删改查 + 状态流转 + 自动归档一气呵成
2. **地图可视化**：高德地图航线绘制 + 无人机移动动画模拟 + 进度实时跟踪
3. **权限控制体系**：基于若依 RBAC，管理员/普通用户权限分离
4. **数据导出能力**：Excel 全模块导出，支持筛选后导出
5. **代码规范**：Controller → Service → Mapper 标准三层架构，关键操作 `@Transactional` 保障

### 仍需关注的事项

| 优先级 | 事项 |
|--------|------|
| 🔴 答辩前 | D1-1 设备状态口径统一（避免评委质疑"离线"状态缺失） |
| 🔴 答辩前 | D1-2 补充 pauseTask 权限注解 |
| 🟡 建议 | D1-3 / D1-4 冗余字段回填 + 级联清理 |
| 🟢 加分 | D3-1 AI 识别增强（答辩高光点） |
| 🟢 加分 | D3-5 巡防报告 PDF 导出 |
