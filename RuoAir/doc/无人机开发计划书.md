### 1. 第一阶段：系统基础“白嫖”与配置

若依已经帮你做好了系统管理模块，你只需要进行简单的“配置”即可满足需求书 3.1 章节的要求 。



- 

  **权限角色配置**：在系统管理中，创建“管理员”和“普通用户”两个角色，严格按照需求配置菜单权限（管理员全权限，普通用户仅查看）。

  

  

- 

  **用户管理与日志**：利用若依自带的用户管理新增测试账号，自带的操作日志模块也完全符合需求 。

  

  

- 

  **清理冗余功能**：隐藏掉若依框架自带但需求书不需要的菜单（如部门管理、岗位管理、定时任务等），保持界面简洁，适配高校答辩风格 。

  

  

### 2. 第二阶段：核心数据库设计与代码生成

这是你接下来要重点做的第一步。你需要设计四个核心业务表，并使用若依的代码生成器一键生成前后端的基础 CRUD（增删改查）代码。

- 

  **无人机设备表 (`sys_uav_equipment`)**：包含设备编号、名称、型号、续航时长、状态等字段 。

  

  

- 

  **巡航航线表 (`sys_uav_route`)**：包含航线名称、预计时长、飞行高度、以及**核心的航线点位 JSON 数据**（用于前端地图渲染）。

  

  

- 

  **巡航任务表 (`sys_uav_task`)**：包含任务名称、关联设备 ID、关联航线 ID、任务状态（待执行、执行中等） 。

  

  

- 

  **巡航结果表 (`sys_uav_result`)**：包含结果编号、巡航概述、发现情况等，并准备好 `@Excel` 注解用于后续的一键导出 。

  

  

### 3. 第三阶段：前端地图与界面深度定制

代码生成后，你需要对 Vue 前端页面进行“魔改”，使其符合业务逻辑。

- 

  **首页大盘（Dashboard）**：修改 `index.vue`，利用 ECharts 和后端统计接口，展示设备、航线、任务的总数概览 。

  

  

- 

  **高德地图集成**：在“航线规划页”引入高德地图 API（`@amap/amap-jsapi-loader`），实现鼠标点击绘制多边形/折线，并获取坐标点组存入数据库 。

  

  

- 

  **地图回显**：在航线详情和结果详情页，通过高德地图展示保存的坐标路线 。

  

  

### 4. 第四阶段：核心业务流转与“模拟”逻辑

这里是这个系统的难点，也是答辩最核心的演示部分，需要脱离纯粹的 CRUD，编写特定的 Service 逻辑。

- 

  **状态联动机制**：编写代码确保当任务点击“开始执行”时，关联的无人机设备状态自动变为“任务中”；任务完成/取消时，设备恢复“正常” 。

  

  

- 

  **巡航动画模拟**：在前端任务页面的“执行中”状态下，利用高德地图的 `Marker` 轨迹回放功能或简单的定时器动画，让无人机图标在地图上沿着航线移动，模拟进度 。

  

  

- 

  **结果自动生成闭环**：当任务点击“完成”时，后端不仅要改状态，还要自动 `INSERT` 一条数据到“巡航结果表”中，实现业务闭环 。

  

  

### 5. 第五阶段：特色与拓展功能

- 

  **图片 AI 识别框取**：在结果详情页，引入 Canvas 或简单的 div 绝对定位，根据模拟的坐标数据在巡航图片上画出“识别框” 。

  

  

- 

  **导出功能**：利用若依自带的 Excel 导出工具类，稍微调整一下导出字段即可完成 。
