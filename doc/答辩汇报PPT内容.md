## 第2页 · 项目背景与目标

**背景**：
- 无人机巡防已广泛应用于水域巡查、电力巡检、违建监测、农业植保等领域
- 巡防任务管理目前多依赖人工记录，信息化程度低，缺乏统一管控平台
- 高校教学中需要一套完整的无人机业务模拟系统，无需依赖真实硬件

**目标**：
- 实现无人机设备 → 航线 → 任务 → 结果的**全流程业务闭环**
- 接入 AI 大模型实现巡防图像的**智能识别与目标框选**
- 系统可本地运行，满足教学演示与答辩要求

## 第3页 · 技术架构

**架构图文字**：

```
┌────────────── 前端 Vue2 + Element UI ──────────────┐
│  设备管理  │  航线规划  │  任务监控  │  结果归档    │
```

## 第6页 · 核心业务流程

**完整业务闭环**：

```
1.录入设备 ──→ 2.绘制航线 ──→ 3.创建任务(绑定设备+航线)
                                        │
                         ┌──────────────┼──────────────┐
                         ▼              ▼              ▼
                      开始任务        取消任务        完成任务
                     设备→任务中      设备→正常      设备→正常
                     进度模拟                      自动生成结果
                         │                              │
                         ▼                              ▼
                      监控界面                    AI识别+框选标注
                    (地图动画+进度)               编辑+PDF/Excel导出
```

**状态机**：待执行(0) → 执行中(1) → 已完成(2) / 已取消(3)

## 第8页 · AI 智能识别（核心亮点）

**技术方案**：
- 接入**阿里千问 VL (Qwen2.5-VL-Max)** 多模态大模型
- 上传巡防图片 → 模型分析图像内容 → 返回结构化 JSON

**识别能力**：
- **目标检测**：识别图像中的车辆、建筑、浮标、垃圾等目标物体
- **精确框选**：返回归一化 bbox 坐标 `[x, y, width, height]`
- **服务端画框**：Java AWT Graphics2D 在图片像素上绘制标注框
- **自动填充**：findings（发现情况）、handling（处理建议）、remark（补充信息）自动填入表单

**处理流程**：上传图片 → Base64编码 → Qwen API → JSON解析 → AWT画框 → 保存标注图 → 返回前端

## 第9页 · 系统亮点与技术难点

**亮点**：
1. 完整业务闭环：设备→航线→任务→结果，四模块联动，状态自动流转
2. AI 大模型集成：千问 VL 实现图像智能分析与目标框选，非简单文字识别
3. 服务端图像标注：AWT 像素级画框，比前端 CSS 叠层更精准
4. 地图可视化：高德地图航线绘制 + 无人机移动动画 + 实时进度跟踪
5. 多格式导出：Excel 批量导出 + iText PDF 报告生成

**技术难点**：
- AI 返回坐标的归一化处理与像素映射
- 事务保障下的任务状态机联动（设备状态同步、结果自动归档）
- 前端地图动画与后端进度调度的数据同步
